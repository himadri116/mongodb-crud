Relationships

One-to-Many & Many-to-Many · Node.js + Mongoose · Library System

📗
One-to-Many

Author → Books

One author writes many books. Each book stores a single author_id reference pointing back to the author.

🏷️
Many-to-Many

Books ↔ Genres

A book can belong to many genres. A genre can tag many books. MongoDB stores an array of genre_ids[] inside each book.

Slide 03 / 05 · How MongoDB Stores It

Document Structure

Unlike SQL, MongoDB stores relationships as references (ObjectIds) inside the document itself. No separate join table is needed for many-to-many.

// ── AUTHOR document (authors collection) ─────────── { _id: ObjectId("64a0000000000000001"), name: "J.R.R. Tolkien", country: "United Kingdom" } // ── BOOK document (books collection) ─────────────── { _id: ObjectId("64b0000000000000001"), title: "The Hobbit", year: 1937, // ONE-TO-MANY: single reference to author author: ObjectId("64a0000000000000001"), ← points to Tolkien // MANY-TO-MANY: array of genre references genres: [ ObjectId("64c0000000000000001"), ← Fantasy ObjectId("64c0000000000000003") ← Adventure ] } // ── GENRE document (genres collection) ───────────── { _id: ObjectId("64c0000000000000001"), name: "Fantasy" }
Slide 04 / 05 · CRUD Operations with Relationships

CRUD Queries

📥 CREATE — Add a book

Stores the author's ObjectId and an array of genre ObjectIds

await Book.create({ title: "Dune", year: 1965, author: herbertId, ← 1 ref genres: [scifiId, adventureId] }) ← array of refs

📖 READ — Get books with author name

.populate() replaces ObjectIds with actual documents (like SQL JOIN)

await Book.find() .populate('author', 'name') .populate('genres', 'name') // Returns full author + genre docs

✏️ UPDATE — Add a new genre to a book

$addToSet safely adds to the genres array without duplicates

await Book.updateOne( { _id: bookId }, { $addToSet: { genres: newGenreId }} )

🗑️ DELETE — Remove genre from ALL books

$pull removes the genre from every book's array at once

await Book.updateMany( {}, { $pull: { genres: deletedGenreId }} )

One-to-many cascade delete: Book.deleteMany({ author: authorId }) — removes all books when an author is deleted.
